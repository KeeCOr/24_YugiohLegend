YugiohLegend UX 및 튜토리얼 개선 대책

현재 게임은 카드/라인/마법 규칙이 짧은 시간에 많이 바뀌었기 때문에, 플레이어가 첫 판에서 무엇을 해야 하는지 즉시 이해하기 어렵다. 이 문서는 가로형 레이아웃 기준으로 UX 문제와 튜토리얼 보강안을 정리한다.

작성일: 2026-05-27 대상: 1600 x 900 가로형 UI 목표: 첫 판 이해도 개선

핵심 문제

목표가 불명확함
플레이어가 첫 화면에서 “카드를 어디에, 왜 내야 하는지”를 바로 알기 어렵다.
행동 가능 여부가 약함
글로우가 있어도 왜 가능한지, 왜 불가능한지 설명이 부족하다.
전투 결과 예측이 어려움
ATK/HP 차이, 라인 해금, 마법 지연 발동이 한 번에 이해되지 않는다.
턴 흐름 안내 부족
1턴은 배치만, 2턴부터 전투라는 핵심 규칙이 플레이 중 계속 보이지 않는다.

UX 개선 방향

영역개선안효과
손패 낼 수 있는 카드는 초록 글로우, 낼 수 없는 카드는 어둡게 처리한다. 마우스 오버 시 “가능/불가능 이유”를 한 줄로 표시한다. 플레이어가 고민해야 할 카드 수가 줄어든다.
라인 잠긴 라인에는 TURN 2 OPEN처럼 해금 턴을 크게 표시한다. 사용 가능 라인은 테두리와 바닥광을 강화한다. 첫 턴에 왜 한 라인만 쓰는지 바로 이해된다.
선택 흐름 카드 선택 후 배치 가능한 라인만 밝게 켠다. 배치 불가능 라인은 클릭해도 이유를 표시한다. 입력 실패가 “버그”처럼 느껴지는 문제를 줄인다.
전투 예측 카드를 라인 위에 올리면 예상 피해, 생존 여부, 직접 공격 여부를 미리 보여준다. ATK/HP 규칙을 설명 없이 체감할 수 있다.
마법 앞면 마법에는 남은 턴 카운터, 뒷면 마법에는 조건 아이콘을 표시한다. 지연 발동과 조건 발동의 차이가 명확해진다.
우측 패널 LP 아래에 현재 턴 목표와 다음 단계 안내를 고정한다. 상태 문구가 사라져도 플레이 방향을 잃지 않는다.

튜토리얼 구조

1단계: 첫 소환

  • 게임 시작 시 손패의 무료 소환 몬스터 1장을 강제로 강조한다.
  • 중앙 라인만 밝게 표시하고, 나머지 라인은 잠금 상태를 유지한다.
  • 안내 문구: “초록색 카드 하나를 선택한 뒤 빛나는 라인에 배치하세요.”

2단계: 커밋

  • 카드 배치 후 COMMIT 버튼에 강한 글로우를 준다.
  • 안내 문구: “배치를 확정하면 상대도 행동하고, 이번 턴은 공격하지 않습니다.”

3단계: 전투 이해

  • 2턴 시작 시 “이제 전투가 발생합니다” 안내를 표시한다.
  • 상대 몬스터와 내 몬스터의 ATK/HP 차이를 전투 전 미리 보여준다.
  • 피해가 발생하면 라인 중앙에 ATK 차이만큼 HP 감소라는 짧은 설명을 띄운다.

4단계: 마법 이해

  • 앞면 마법을 배치하면 라인 위에 1T 카운터가 붙는 것을 보여준다.
  • 뒷면 마법은 “조건이 맞으면 공개”라는 안내와 함께 뒤집힌 카드로 표현한다.

5단계: 제물 소환

  • 제물 몬스터가 손패에 있을 때 필요한 제물 수를 크게 표시한다.
  • 제물이 부족하면 카드 오버 시 “제물 1장이 필요합니다”를 표시한다.
  • 제물이 충분하면 자동 제물 후보 라인을 노란색으로 표시한다.

화면별 안내 문구

상황추천 문구
게임 시작초록색으로 빛나는 카드는 지금 낼 수 있습니다.
라인 잠김 클릭이 라인은 아직 잠겨 있습니다. 다음 턴에 열립니다.
몬스터 선택빈 라인에 소환할 수 있습니다. 제물이 필요한 카드는 조건을 먼저 확인하세요.
마법 선택마법은 라인에 설치되고, 조건이나 턴이 맞으면 발동합니다.
커밋 가능배치가 끝났다면 COMMIT을 눌러 턴을 진행하세요.
커밋 후상대 행동을 기다리는 중입니다.
전투 시작정면 전투는 ATK 차이만큼 HP를 잃습니다.
직접 공격상대 라인이 비어 있어 LP에 직접 피해를 줍니다.

우선순위

순위작업이유
1손패 가능/불가능 사유 툴팁현재 가장 큰 혼란은 “왜 안 되는지 모름”이다.
2카드 선택 후 배치 가능 라인 강조플레이 입력을 자연스럽게 유도한다.
3첫 판 튜토리얼 오버레이첫 판 이탈을 줄인다.
4전투 예측 UI규칙 이해와 전략성을 동시에 높인다.
5마법 카운터/조건 아이콘지연 마법과 뒷면 마법의 정체성을 강화한다.
가장 먼저 할 일은 튜토리얼 전체를 크게 만드는 것이 아니라, 매 입력 순간에 “가능한 행동”과 “불가능한 이유”를 바로 보여주는 것이다.

구현 체크리스트

  • 손패 카드 상태: playable, blocked, selected, queued 4단계로 분리한다.
  • 카드 오버 시 상태 문구를 우측 패널 또는 하단 상태 영역에 표시한다.
  • 선택 카드 기준으로 라인 하이라이트를 재계산한다.
  • 처음 1판만 튜토리얼을 자동 표시하고, 이후에는 설정에서 다시 볼 수 있게 한다.
  • 튜토리얼 중에도 실제 게임 규칙을 그대로 사용한다.
